ID Churn Sentinel

Technical alpha: a candidate-source registry and human-review pipeline for US government pages about name and gender-marker changes on identity documents. When operational, it reports that an eligible source page changed and shows the passage that changed. It never asserts what the law is.

Do not rely on this list as authoritative guidance, and do not hand it to someone who will. A machine cannot tell a state's real birth-certificate page from a convincing-looking one. It cannot even reliably tell a live page from a bot-wall: one federal site answers our crawler with a page titled Request Access and serves it with HTTP 200. A status code is not a verification, and a title is not a verification. A person is.

What this tool does claim

  • For a published observation: this URL changed — a successful eligible watch found text different from its retained baseline.
  • For a published text observation: this is what changed in it — the passage, with a reproducible hash before and after.

What this tool never claims

  • What the law is, or what any change means legally. Not in the feed, not in a summary field, not ever.
  • That an unverified URL is the right page. Until a named human has confirmed it, the entry is a lead, not a citation.

Every source below, in sources.json, and in every per-jurisdiction feed carries a machine-readable verification_status field for exactly this reason — so that an integrator cannot consume a source without being told what is behind it. The verification queue and how it is worked: docs/VERIFYING.md.

What this tool will and will not tell you

It will tell you

  • For a published observation, that an eligible registered page changed, with the passage that changed, its URL, and a reproducible hash of the page text before and after.
  • That an eligible page stopped answering for long enough that an outage is no longer the likeliest explanation.
  • Exactly which candidates are not monitored, and why.

It will never tell you

  • What the law is. A change to a web page is not a change to the law, and a hash comparison cannot read law. Advocates for Trans Equality, Trans Lifeline, Namesake and lawyers do that work; this is the monitor underneath it, not a replacement for it.
  • What a change means. Whether a change is editorial or substantive is a judgment, and a named human makes it. Nothing here is classified by a machine.
  • What you should do. This is not legal advice.

Silence from this feed is not evidence that nothing changed. Policy can move by an internal directive that never touches a web page, and this tool would see nothing. Read the gaps below before you read the silence.

Endpoints

No account. No API key. No email address. No tracking. Fetch these with curl, a feed reader, or a cron job. We do not want to know who reads this: a subscriber list for a trans-ID-law feed is a list of trans people, and the only way to keep that list safe is to never create it. Consequently we cannot report readership and never will.

  • feed.xml — RSS 2.0, every jurisdiction.
  • changes.json — the versioned JSON feed. This is the one you integrate against.
  • sources.json — the full inventory of registered candidates, exact attempt eligibility, and gaps, so you can map your own pages to our source_ids.
  • status.json — persisted watcher health, exact attempt denominator, and last successful run. Its time is independent of page generation.

What is NOT watched, and why

There are 12 named gaps. The feed's silence about any of them means nothing at all. Each one is a refusal we chose: in every case there is a two-line change — spoof a browser User-Agent, disable certificate verification, ignore a robots.txt — that would close the gap today and make this a tool that lies to a government server, about who it is, on behalf of a population under surveillance. We do not make it. The gap is recorded instead.
