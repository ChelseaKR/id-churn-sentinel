# Internationalization declaration

**I18N-Standard: Applies**

**Owner:** Accessibility and language lead

**Current state:** English-only technical alpha

**V1 scope:** reviewed Spanish navigation, status definitions, limitations,
correction/reporting instructions, and integration basics

**Target date:** 2026-11-13, before the V1 release-candidate evidence window

The canonical source evidence and government text remain in the language
published by the authority. The product must not machine-translate statutes,
source content, diffs, reviewer judgments, or legal terms and present that
output as authoritative.

Implementation and release requirements live in
[`08-ACCESSIBILITY-I18N.md`](./08-ACCESSIBILITY-I18N.md). The required workflow
is stable message IDs, a Spanish translator, an independent reviewer familiar
with trans and legal-information language, accessibility QA, and a fail-closed
staleness check. When an approved Spanish string is stale, the surface falls
back to clearly labelled English; it never serves an outdated translation as
current.

This declaration closes the ownership and scope ambiguity only. It does not
claim that catalogs, translations, reviewer sign-off, or the stale-string gate
already exist. Those remain V1 release gates in
[`15-V1-RELEASE-CHECKLIST.md`](./15-V1-RELEASE-CHECKLIST.md).
